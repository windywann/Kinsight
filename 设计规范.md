# Kinsight 平台 UI/UX 设计规范 (AI 原型生成指南)

## 1. 总体设计原则与风格 (Design Philosophy)
*   **产品定位**：企业级数据分析、算法建模与因果推断平台 (B端 SaaS)。
*   **设计风格**：典型的现代企业级中后台管理系统。极简、高效、数据导向。高度疑似基于 Ant Design 或 Element Plus 等成熟 UI 框架定制。
*   **视觉特征**：大面积留白，通过浅灰色背景与白色卡片区分层级，主色调为品牌蓝。界面无过多修饰，强调用排版和间距划分信息。
*   **注（AI 忽略项）**：截图中存在斜向排列的半透明水印（名字+日期），这是系统安全水印，**在生成新页面原型时请一律忽略，不要在 UI 中体现水印。**

## 2. 色彩系统 (Color Palette)
在生成 UI 或 CSS 时，请遵循以下语义化色彩：
*   **品牌/主色 (Primary)**: 经典科技蓝 (近似 `#1677FF` 或 `#409EFF`)。用于主按钮、Tab 激活态、链接、进度条、选中边框。
*   **成功 (Success)**: 绿色 (近似 `#52C41A`)。用于成功状态图标、完成标签。
*   **危险/错误 (Danger/Error)**: 红色 (近似 `#F5222D`)。用于删除按钮文本、失败状态、必填项星号 `*`。
*   **文字色 (Typography)**:
    *   主要文字 (标题、正文): 深灰黑 (近似 `#333333` 或 `rgba(0,0,0,0.85)`)
    *   次要文字 (表头、说明、占位符): 中性灰 (近似 `#999999` 或 `rgba(0,0,0,0.45)`)
*   **背景与边框 (Background & Border)**:
    *   页面大背景色: `#f8fafb`
    *   内容区/卡片背景: 纯白 (`#FFFFFF`)
    *   分割线/表格边框: 浅灰 (`#E8E8E8` 或 `#EBEEF5`)
    *   侧边栏选中态背景: 极浅蓝 (带有品牌色 10% 左右透明度的背景)

## 3. 全局布局 (Global Layout)
采用标准的“左侧导航 + 顶部顶栏 + 主内容区”布局。

### 3.1 左侧菜单栏 (Sidebar)
*   **顶部 Logo 区**: 文本 "Kinsight"，**不加粗 (font-weight: 400)**，字号 `20px`。
*   **菜单项 (nav-item)**:
    *   支持一级和二级菜单（折叠/展开）。
    *   **字体与颜色**：字号统一为 `14px`。默认态文本颜色为 `#252626`。
    *   **悬浮态 (Hover)**：文本及图标颜色变为浅蓝色 `rgba(50, 107, 251, 0.8)`。
    *   **选中态 (Active)**：文本及图标颜色变为主色蓝 `#326bfb`。
*   **二级菜单 (sub-item) 尺寸与边距**：
    *   Padding: `14px 18px 14px 58px`。
    *   宽度：自适应或撑满容器，不设固定宽度和圆角。
    *   选中态背景：浅蓝色底 `rgba(50, 107, 251, 0.08)`，字体加粗为 `500`。
*   **任务列表页导航映射（新增）**：
    *   一级菜单顺序及 Icon：
        *   `首页` (home.svg)
        *   `Uplift建模` (list.svg) -> 二级：`项目管理`、`人群包管理`
        *   `因果发现` (wallet.svg) -> 二级：`项目管理`
        *   `端智能建模` (home.svg) -> 二级：`项目管理`、`eve包管理`
    *   `task-list-demo.html` 归属路径：`端智能建模 / 项目管理`。

### 3.2 顶部顶栏 (Header)
*   纯白背景，高度固定（约 56px - 64px）。
*   左侧区域留白（通常配合面包屑，但本系统主要在内容区内展示层级）。
*   右侧操作区：包含带图标的快捷链接（如“用户群”、“用户文档”）+ 用户头像（圆形）。

### 3.3 主内容区 (Main Content)
*   背景色为浅灰色。
*   **内容卡片 (Card)**: 所有的核心内容（筛选区、表格、详情表单）均包裹在纯白背景、带轻微圆角（通常为 4px-8px）的无阴影卡片容器中。

## 4. 核心 UI 组件规范 (Component Library)

### 4.1 按钮 (Buttons)
*   **主按钮 (Primary Button)**: 蓝色背景，白色文字，无边框。通常位于页面右上角（如“新建项目”、“新建任务”）或弹窗的“提交”动作。
*   **次级/默认按钮 (Default Button)**: 白色背景，浅灰色边框，黑色文字。如筛选区的“清除筛选”、弹窗的“取消”。
*   **文字按钮/链接按钮 (Text/Link Button)**: 无背景，主色（蓝）或危险色（红）文字。多用于表格的“操作”列（如详情、删除、复制）。

### 4.2 表单与筛选区 (Forms & Filters)
*   **布局**：筛选区通常在列表页上方，采用 inline 布局（水平排列）。新建/详情表单常采用上下排布或左右（Label: Value）排布。
*   **输入框 (Input)**: 矩形，浅灰色边框，内带灰色 Placeholder（如“请输入项目名称”）。
*   **复选框/单选框 (Checkbox/Radio)**: 紧跟文字，如“我收藏的”、“我负责的”。
*   **必填项**：Label 前带有红色星号 `*`。

### 4.3 表格 (Tables)
*   **表头 (Table Header)**: 灰色底色（非常浅），字体加粗，黑色。
*   **数据行 (Row)**: 白色背景，行与行之间有浅灰色底边框。
*   **收藏交互**：ID 旁边常跟一个五角星图标（灰色表示未收藏，点亮表示收藏）。
*   **操作列**：固定在最右侧，操作项平铺（文字链接），“⋮”点击后展示白底圆角下拉菜单（阴影浮层），菜单项为“详情 / 复制 / 删除”，其中“删除”为红色。
*   **头像组件**：负责人列采用“kim 图标 + 姓名”的组合，图标 URL 固定为 `https://w2.kskwai.com/kos/nlav12127/svg-icons/kim.svg`。

### 4.4 状态指示与标签 (Status Indicators & Tags)
*   **文字标签 (Tag)**: 矩形，带边框和极浅的背景色。例如：“审批中”（蓝字蓝框）、“导入数据”（蓝字蓝框）。
*   **图标+状态文字**:
    *   绿色勾图标 + "已完成"
    *   红色叉图标 + "失败"
*   **步骤流程卡片 (Step Cards)**: （重点复用，见图3）用于展示任务执行阶段。
    *   独立的白色卡片，包含阶段名（如“预处理”、“训练”）、状态图标、开始/结束时间、耗时等。
    *   卡片之间用双箭头 `>>` 连接表示流转方向。
    *   若包含“成功/失败/运行中/等待中”统计，**仅数字上色**，标签文字保持默认色：
        *   成功数字：`#67c23a`
        *   失败数字：`#f56c6c`
        *   运行中数字：`#409eff`
        *   等待中数字：`#e6a23c`

### 4.5 导航与容器 (Navigation & Containers)
*   **面包屑 (Breadcrumb) / 返回头**: `< 任务详情`，加粗大字号，右侧常伴随“负责人”信息和操作按钮。
*   **标签页 (Tabs)**:
    *   样式：文字排布，**选中态底部有主色蓝色粗横线**（如下划线），非选中态为灰色文字。
    *   应用场景：详情页中切换不同维度的数据（如：子任务详情、评估报告、预测结果）。
*   **分段控制器 (Segmented Control)**: （见图5）作为筛选器出现，白底，选中项变成蓝框蓝字。

### 4.6 弹窗 (Modals/Dialogs)
*   居中显示，包含黑色半透明遮罩层。
*   **Header**: 标题居左，关闭图标 `X` 居右。
*   **Body**: 放置表单项（带有 `*` 必填标识）。
*   **Footer**: 右下角放置操作按钮（左“取消”，右“提交工单”主按钮）。

### 4.7 数据可视化 (Data Visualization)
*   使用轻量级的图表风格（类似 Echarts）。
*   **图表工具栏**：支持图表内部的分页（如 `< 1/2 >`）、图例。
*   **结构图/网络图**：支持通过滑块 (Slider) 控制阈值，支持下拉框选择布局方式，支持全屏、重置视图等顶部操作栏。

## 5. 标准页面模板结构 (Page Templates)

当要求你生成新功能时，请优先套用以下两种页面结构：

### 模板 A：标准列表页 (List / Index Page)
```text
[Header 区域] (面包屑/标题) + [主操作按钮(最右)]
-----------------------------------------------------------
[白底卡片容器 开始]
  [筛选区域 (Inline Form)]
    - 输入框 1 | 输入框 2 | 复选框 | [查询(蓝)] [重置(白)]
  ---------------------------------------------------------
  [表格区域 (Table)]
    - 列表头
    - 数据行 (带状态标签和文字操作链)
    - 底部翻页器 (Pagination - 虽然截图未完全展示，但必须默认包含)
[白底卡片容器 结束]
```

**模板 A 字号与间距补充规范（与任务列表截图对齐）**
* 页面主标题（如 `<h1 class="title">任务列表</h1>`）：`20px`
* 筛选区字段标题（如 `.f-label` 的“任务名称 / 负责人”）：`14px`
* 卡片头部区域（如 `div.header`）：上下 padding 为 `20px`（即 `padding: 20px 20px;` 或根据需要设置左右值）。

### 模板 B：详情/工作台页面 (Detail / Workspace Page)

```text
[Header 区域] < 返回按钮 + 标题名称 | 基础属性展示 | [页面级操作按钮]
-----------------------------------------------------------
[白底卡片容器 1：基础信息/执行流] (可选，通常可折叠 `^`)
  - Key-Value 属性平铺
  - 流程步骤卡片 (如适用)
---------m j--------------------------------------------------
[Tab 切换区] Tab 1 (激活态底边框) | Tab 2 | Tab 3
-----------------------------------------------------------
[白底卡片容器 2：Tab 具体内容]
  - 可能是图表
  - 可能是二级筛选器 + 数据回测表格
  - 可能是网络关系图 (带独立控制栏)


# AI 角色指令 (Prompt Instructions for AI)
当用户基于此规范要求你生成代码或设计时：

1. 优先使用框架标准组件：假设前端栈为 React + Ant Design 或 Vue + Element Plus，请直接使用这些框架的标准组件名（如 Table, Form, Button type="primary", Tabs）来构建 UI 原型。
2. 遵守空间一致性：保持适当的 padding 和 margin，模仿中后台系统的疏朗感。
3. 不用重新发明轮子：遇到“状态”、“操作”等概念时，直接使用规范中定义的 Tag、Text Button 等样式。
4. 输出格式：根据用户需求，可输出 React/Vue 代码、HTML/Tailwind 结构，或用 Markdown 绘制详细的 ASCII 界面线框图。
